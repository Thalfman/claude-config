# Package Contents — One-Paragraph Reference

Each `(JB, entity)` folder contains six items, ready for direct submission:

1. **Cover letter** (`cover_letter.pdf`) — 3-4 sentences on letterhead, signed by Thomas Halfman.
2. **Application form** (`application_form.pdf`) — entity's published form, every field filled or marked `(pending — see open_items.md)`.
3. **Construction drawings** (`construction_drawings.pdf`) — scrubbed CD (state-correct 811, correct legal block).
4. **Route map / KML** (`route.kml`) — copied from input KML when available; Mode C jobs may omit.
5. **Fee calculation** (`fee_calculation.pdf`) — line items + total + fee schedule citation.
6. **Transmittal** (`transmittal.pdf`) — single-page index. The COI line is a one-line broker reference with the additional-insured wording inline. The TCP line is a one-line MUTCD-typical reference (or "custom TCP required" when the work needs designed traffic control).

Entity-specific reference docs (e.g., `reference_MDOT_Form_2205-1_General_Conditions.pdf`) are appended at the end of `SUBMISSION_PACKAGE.pdf`.

What is **not** in the package:
- Separate `coi.md` / `coi.pdf` — the COI is broker-issued; the additional-insured wording lives on the transmittal line.
- Separate `route_map.md` / `route_map.pdf` — the KML is the route exhibit; render a PDF only if the entity insists on a paper exhibit.
- Separate `traffic_control_plan.md` — only generate when work needs custom design (lane closure, detour, full closure).
- `research.md` — entity contact info goes on the cover letter and the transmittal; verification calls go in `open_items.md`.
- `qc_report.md` — the skill verifies during generation; outputs ship clean.
- Project-level `open_items.md` — only one open-items file per JB.

Drafting rules per item are in the corresponding section of `SKILL.md`.
